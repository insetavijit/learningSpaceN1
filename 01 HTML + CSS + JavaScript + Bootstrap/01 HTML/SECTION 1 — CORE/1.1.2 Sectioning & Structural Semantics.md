## **Topics Table — Sectioning & Structural Semantics (HTML5)**

| **Element / Concept**         | **Brief Description**                                                                                                                 |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `<header>`                    | Introductory container for a page or section, holding headings, branding, or metadata; defines contextual entry, not layout position. |
| `<footer>`                    | Concluding container for a page or section, containing authorship, legal info, related links, or meta content tied to its parent.     |
| `<main>`                      | Represents the dominant, unique content of the document; single per page and central for accessibility navigation.                    |
| `<section>`                   | Thematic grouping of related content, usually with a heading; contributes to the logical outline of the document.                     |
| `<article>`                   | Self-contained, independently distributable unit of content such as posts, cards, or comments; meaningful outside its context.        |
| `<nav>`                       | Major navigation block that provides wayfinding through the site or document; reserved for primary link groups.                       |
| `<aside>`                     | Complementary or tangential content related to its surrounding context, such as sidebars, callouts, or related resources.             |
| Document outline algorithm    | Conceptual model where sectioning elements and headings form a hierarchical structure describing the document’s topics.               |
| Heading hierarchy (`h1`–`h6`) | Ordered headings that title sections and communicate structure to users and assistive technologies.                                   |
| Landmark semantics            | Mapping of semantic elements to accessibility landmarks enabling region-based navigation for screen readers.                          |
| [[Sectioning vs grouping]]    | Distinction between semantic sectioning (`section`, `article`) and non-semantic grouping (`div`, `span`).                             |
| Nesting rules                 | Valid structural relationships among sectioning elements to maintain a coherent content hierarchy.                                    |
| Accessibility impact          | How proper semantics improve screen reader navigation, skip links, and overall usability.                                             |
| SEO implications              | Search engines use semantic structure to better interpret content importance and topical hierarchy.                                   |
| Common misuse patterns        | Anti-patterns such as overusing `<section>`, misusing `<article>`, or redundant landmarks that degrade meaning.                       |
