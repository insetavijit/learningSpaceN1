## **[[1.4.1 Parsing & DOM Construction]]**

### **Objective**

Build a precise mental and practical model of how browsers parse HTML, recover from errors, and construct the DOM tree that becomes the foundation for rendering and scripting.

### **Approach**

Treat parsing as both a **streaming process (bytes → tokens → nodes)** and a **structural synthesis step (document → DOM tree)**. Study it from the dual perspective of **spec-defined algorithms** and **observable browser behavior**.

### **Method**

- Trace **HTML tokenization**: how the parser converts incoming characters into tags, attributes, and text tokens.
    
- Observe **DOM tree creation** as tokens are transformed into element, text, and comment nodes.
    
- Experiment with malformed markup to study **error recovery mechanisms** and how browsers normalize broken HTML.
    
- Study **incremental parsing**, where the DOM is built progressively as bytes arrive over the network.
    
- Inspect the live DOM in DevTools while a page is loading to observe node insertion order.
    
- Compare `document.readyState`, `DOMContentLoaded`, and load events to understand parsing milestones.
    
- Insert scripts at different positions to see how they block or interleave with parsing.
    
- Modify markup dynamically and observe how it updates the DOM tree.
    

### **Constraints**

- Do not assume browsers reject invalid HTML; they repair and continue.
    
- Avoid relying on malformed markup to produce specific DOM structures.
    
- Do not manipulate DOM nodes before they exist in the parsed tree.
    
- Be aware that synchronous scripts can block parsing.
    
- Avoid layouts that depend on parser error recovery quirks.
    
- Do not confuse source HTML with the normalized DOM produced by the browser.

|**Topic**|**Brief Description**|
|---|---|
|**HTML tokenization**|The process by which the browser’s HTML parser reads raw markup and converts it into a stream of tokens (start tags, end tags, text, comments) according to the HTML parsing algorithm.|
|**DOM tree creation**|The transformation of tokens into a hierarchical Document Object Model, where elements become nodes linked in parent–child relationships that represent the document’s structure in memory.|
|**Error recovery mechanisms**|Built-in parsing rules that handle malformed or invalid HTML by inferring missing tags and restructuring nodes, ensuring pages still render predictably across browsers.|
|**Incremental parsing**|The browser’s ability to parse and build the DOM progressively as bytes arrive from the network, enabling early rendering and script execution before the full document is downloaded.|
