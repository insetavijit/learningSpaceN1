## **HTML Document Skeleton — Topics Table**

| **Element / Concept**       | **What to Learn**                                            | **Why It Matters**                                             |
| --------------------------- | ------------------------------------------------------------ | -------------------------------------------------------------- |
| `<!DOCTYPE html>`           | Purpose of DOCTYPE, HTML5 standard mode, quirks vs standards | Ensures browsers render the page using modern HTML5 rules      |
| `<html>` root               | Root element role, nesting rules, global attributes          | Wraps the entire document and defines the HTML context         |
| `lang` attribute            | Setting document language (e.g., `en`, `en-IN`)              | Improves accessibility, SEO, and screen reader behavior        |
| `<head>` section            | Metadata container, what belongs in head vs body             | Holds non-visible data that configures the document            |
| `<meta charset>`            | UTF-8 encoding, placement in head                            | Prevents text corruption and ensures correct character display |
| `<meta viewport>`           | Responsive viewport settings for mobile                      | Enables proper scaling on mobile devices                       |
| `<title>`                   | Page title rules, length, uniqueness                         | Appears in browser tabs and search engine results              |
| `<meta name="description">` | SEO description usage and best practices                     | Influences search snippets and click-through rate              |
| `<link>`                    | Linking CSS, icons, external resources                       | Connects stylesheets and site assets                           |
| `<script>` (in head)        | Blocking vs deferred scripts, `defer`, `async`               | Controls script loading without hurting performance            |
| `<base>`                    | Base URL and target behavior                                 | Simplifies relative URLs across the document                   |
| `<body>`                    | Content container, visible DOM scope                         | Holds all renderable UI and interactive elements               |
| Script placement (body)     | End-of-body scripts, performance patterns                    | Avoids render blocking and speeds up load                      |
| Comment structure           | HTML comments, documentation habits                          | Improves maintainability and team clarity                      |
| Minimal valid skeleton      | Writing a complete boilerplate from memory                   | Foundation skill for every HTML page                           |

## **Canonical HTML5 Skeleton Example**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document Title</title>
  <meta name="description" content="Brief page description">
  <link rel="stylesheet" href="styles.css">
</head>
<body>

  <!-- Page content goes here -->

  <script src="app.js" defer></script>
</body>
</html>
```
