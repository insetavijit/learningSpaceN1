### **1. Eleventy (11ty)**

###### **Eleventy (11ty)** is a JavaScript-based static site generator that compiles content, data, and templates into static HTML files at build time, producing deployable websites without requiring a runtime server.

**What it is:** A flexible, content-first build tool for generating clean, maintainable, and high-performance static websites using multiple templating languages within modern Jamstack workflows.

**What it is not:** Not a client-side JavaScript framework, not a backend web server, not a full-stack platform, and not a runtime rendering system.

**Summary (Easy Language):**  
Eleventy builds fast websites from files before you publish them, but it does not run apps or servers.



### **2. Static Site Generator (SSG)**

###### **Static Site Generator (SSG)** is a class of tools that transform source content and templates into static HTML files at build time for direct deployment to static hosts.

**What it is:** A build-time publishing system that produces pre-rendered pages for fast delivery, simplicity, and reliability, independent of server-side processing.

**What it is not:** Not a dynamic server-rendered system, not a client-side SPA framework, and not dependent on databases or runtime page generation.

**Summary (Easy Language):**  
An SSG makes web pages ahead of time so they can be served quickly without a server doing work.



### **3. Build-Time Rendering**

###### **Build-Time Rendering** is the process of generating complete HTML pages during a build step rather than creating them dynamically when users request them.

**What it is:** A compilation approach where content and templates are rendered into final documents before deployment, ensuring predictable output and high performance.

**What it is not:** Not runtime rendering, not on-demand server generation, and not client-side rendering in the browser.

**Summary (Easy Language):**  
Build-time rendering creates pages before users visit, so sites load faster.
