###### Q1. What is the primary **purpose** of version control systems?

=> track and manage change history
###### Q2. What does a version control system mainly **track**?

=> file versions over time
###### Q3. Why are version control systems **important for teams**?

=> enable collaboration and rollback safely
###### Q4. What defines a **centralized** version control system?

=> single shared central repository
###### Q5. What defines a **distributed** version control system like Git?

=> full local copies of repository

###### Q6. Name one example of a **centralized** version control system.

=> Subversion SVN system

###### Q7. Name one **distributed** version control system besides Git.

=> Mercurial distributed version control

###### Q8. What internal **model** does Git use to store history?

=> stores complete content snapshots

###### Q9. What kind of data does Git **avoid storing**?

=> avoids storing file deltas

###### Q10. How are **commits treated** in Git’s design?

=> immutable content addressable objects

###### Q11. Why are Git **branches lightweight** and fast?

=> branches are simple commit pointers

###### Q12. Why are most Git operations **faster than SVN**?

=> operations happen locally without server

###### Q13. What drives Git’s **widespread adoption** today?

=> strong ecosystem and tooling support

###### Q14. What are the three main **areas** in Git workflow?

=> working directory staging repository

###### Q15. In which Git area do developers **edit files**?

=> working directory tree

###### Q16. In which area are changes **prepared before commit**?

=> staging area index

###### Q17. In which area are commits **finally stored**?

=> local Git repository

###### Q18. What are the basic Git **file states**?

=> untracked modified staged committed

###### Q19. How is Git commonly **installed on Linux** systems?

=> install using package manager

###### Q20. How is Git commonly **installed on Windows** systems?

=> install using official installer

###### Q21. Which command **verifies** Git installation?

=> git --version command

###### Q22. Which command is used to **edit configuration**?

=> git config command

###### Q23. What user **identity** must be configured first?

=> user name and email

###### Q24. Which config key stores the **author name**?

=> user.name configuration key

###### Q25. Which config key stores the **author email**?

=> user.email configuration key

###### Q26. Which config key sets default **text editor**?

=> core.editor configuration key

###### Q27. Which setting controls **line ending conversion**?

=> core.autocrlf setting

###### Q28. Which **protocol** uses key pairs for Git access?

=> secure shell SSH protocol

###### Q29. Which command **generates SSH keys** locally?

=> ssh-keygen command

###### Q30. Which file stores the SSH **private key**?

=> id_rsa private key file

###### Q31. Which tool stores HTTPS **credentials securely**?

=> Git credential helper

###### Q32. Name one popular Git **hosting platform**.

=> GitHub GitLab or Bitbucket

###### Q33. What links local Git to **remote platform**?

=> SSH public key

###### Q34. Which file stores global Git **configuration**?
=> ~/.gitconfig global configuration file

###### Q35. Which feature allows **including other configs**?
=> include directive in config
