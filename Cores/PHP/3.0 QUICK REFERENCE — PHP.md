# **3.5 Error & Issue Playbook**

**Section Overview (45–65 words)**  
A troubleshooting quick-reference guide that turns common problems into structured, repeatable debugging strategies. Understanding root causes prevents repeated frustration and reinforces engineering precision.

|Error|Cause|Fix|
|---|---|---|
|**Undefined index/variable**|Accessing missing value|Defensive checks `isset`, `??` null coalescing|
|**Headers already sent**|Output before header or session calls|Remove whitespace/output before headers|
|**Call to undefined function/method**|Function missing or autoload failure|Check namespace, import, run `composer dump-autoload`|
|**Class not found**|Namespace mismatch|Align PSR-4 path and ensure correct case|

# **3.6 Best Practices**

**Section Overview (50–75 words)**  
Guidelines to enforce quality, safety, reliability, and maintainability. These principles represent hard-earned lessons from real production systems and should guide decision-making as technical debt compounds quickly in PHP without discipline.

---

## **3.6.1 Do’s & Don’ts**

|Principle|Explanation|
|---|---|
|**DO use prepared statements**|Prevent SQL injection and guarantee data safety.|
|**DO enable strict typing**|Improves predictability and reduces debugging complexity.|
|**DO centralize configuration**|Avoid hard-coded credentials or magic strings.|
|**DON’T trust user input**|Validate and sanitize everything.|
|**DON’T mix HTML, SQL, and logic**|Keep architecture layered to prevent chaos.|

---

## **3.6.2 Performance Guidelines**

|Recommendation|Benefit|
|---|---|
|**Use Opcache**|Eliminates unnecessary recompilation cycles.|
|**Reduce query count**|Avoid loops containing DB calls.|
|**Cache expensive data**|Redis/Memcached reduce load.|
|**Reuse PDO prepared statements**|Major performance win for batch operations.|

---

## **3.6.3 Security Considerations**

|Requirement|Value|
|---|---|
|**Validate input & escape output**|Prevents XSS and injection attacks.|
|**CSRF protection**|Stop forged form submissions.|
|**Secure session + cookie flags**|Protect authentication and identity.|
|**Safe file uploads**|Allowlist MIME, unique filenames, quarantined storage.|
